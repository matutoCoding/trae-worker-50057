## 1. 产品概述

殡仪馆遗体接运调度系统是一款面向殡仪馆接运班组的专业调度管理软件，用于规范化管理遗体接运全流程，包括电话受理、车辆调度、人员排班、路线导航、交接登记、冷藏管理和统计分析七大核心模块，提升接运效率与服务质量。

- 目标用户：殡仪馆接运班组、调度员、接运司机、殡仪服务人员
- 核心价值：实现接运业务全流程数字化管理，提升调度效率，规范服务标准

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 调度员 | 负责电话受理和任务派发 | 全部模块操作权限 |
| 接运司机 | 负责执行接运任务 | 任务查看、导航、交接登记 |
| 管理员 | 负责系统配置和统计 | 全部权限 + 系统设置 |

### 2.2 功能模块

1. **接运任务**：电话受理登记、出车任务派发、接运记录查询
2. **车辆调度**：灵车状态监控、车辆信息管理、车辆排班
3. **人员排班**：接运人员排班、夜间值班安排、人员信息管理
4. **路线导航**：接运地址导航、最优路线规划、里程统计
5. **交接登记**：医院殡仪交接单、家属信息核对、签字确认
6. **冷藏管理**：遗体冷藏柜位管理、停尸登记、柜位状态监控
7. **统计分析**：接运量统计、里程统计、工作效率分析

## 3. 核心流程

### 3.1 接运任务流程

```mermaid
flowchart LR
    A["电话受理"] --> B["任务登记"]
    B --> C["调度派发"]
    C --> D["出车接运"]
    D --> E["到达地点"]
    E --> F["交接登记"]
    F --> G["运回殡仪馆"]
    G --> H["冷藏入库"]
    H --> I["任务完成"]
```

### 3.2 主要用户流程

1. 调度员接到家属电话，登记逝者信息和接运地址，创建接运任务
2. 调度员根据车辆和人员状态，派发任务给指定车辆和人员
3. 接运司机接收任务，使用导航功能前往接运地点
4. 到达后与家属/医院办理交接手续，填写交接单并核对信息
5. 将遗体运回殡仪馆，办理冷藏入库登记
6. 系统自动统计接运数据和里程信息

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深海蓝 (#1e3a5f) - 代表庄重、专业、可靠
- **辅助色**：金色 (#c9a962) - 代表尊重、庄重
- **背景色**：浅灰 (#f5f6f8) - 简洁干净
- **强调色**：红色 (#e53935) - 用于紧急状态和重要提醒
- **按钮风格**：圆角矩形，悬停有轻微阴影和颜色变化
- **字体**：思源黑体 / Noto Sans SC，清晰易读
- **布局风格**：左侧导航 + 顶部状态栏 + 主内容区，卡片式布局
- **图标风格**：线性图标，简洁专业

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 工作台 | 首页概览 | 数据卡片、任务列表、车辆状态网格、快捷入口 |
| 接运任务 | 任务管理 | 顶部筛选栏、任务卡片列表、状态标签、操作按钮 |
| 车辆调度 | 车辆管理 | 车辆状态看板、车辆卡片、分配弹窗 |
| 冷藏管理 | 柜位管理 | 柜位平面图、状态颜色标识、停尸信息弹窗 |
| 统计分析 | 数据报表 | 折线图、柱状图、饼图、数据表格 |

### 4.3 响应式设计

- 桌面端优先设计（1920px基准）
- 支持平板尺寸自适应（1024px以上）
- 侧边栏可折叠，优化大屏和小屏体验

### 4.4 交互体验

- 任务状态以不同颜色标识，一目了然
- 重要操作有确认弹窗，防止误操作
- 数据加载有骨架屏效果
- 表单填写有实时校验提示
- 状态变更有平滑过渡动画
